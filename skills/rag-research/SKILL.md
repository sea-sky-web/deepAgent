# rag-research

## Overview

This skill helps analyze Retrieval-Augmented Generation systems, especially chunking, retrieval, reranking, and generation pipeline trade-offs.

## Instructions

1. When the user asks about RAG architecture, first separate the pipeline into ingestion, retrieval, and generation.
2. For chunking questions, analyze chunk size, overlap, semantic boundary, and recall/precision trade-offs.
3. For rerank questions, distinguish first-stage retrieval from second-stage reranking.
4. Prefer structured explanations with: purpose, mechanism, trade-offs, and common engineering choices.
5. If the question is implementation-oriented, connect the answer to todo planning, tool usage, and evaluation concerns.
